1. INTRODUCTION                                 *notebook_style-introduction*

notebook_style.nvim is a Neovim plugin that renders Python file cells
(separated by # %% delimiters) with beautiful, customizable borders.

Features:
  - Visual cell borders on all four sides
  - Cell names displayed from delimiters (e.g., # %% My Cell Name)
  - Smart visibility (hides delimiters in normal mode, shows labels in borders)
  - Fully customizable colors and border styles
  - Non-intrusive rendering using native extmarks
  - Experimental inline Python cell execution through a Rust Jupyter backend
  - .ipynb notebook workflow through Jupytext-backed # %% Python buffers
  - Lightweight and efficient

2. INSTALLATION                                 *notebook_style-installation*

Using lazy.nvim: >lua
    {
      'stellarjmr/notebook_style.nvim',
      ft = 'python',
      opts = {},
    }
<

With the optional inline execution backend installer: >lua
    {
      'stellarjmr/notebook_style.nvim',
      version = 'v0.7.0',
      ft = 'python',
      build = function(plugin)
        local install = loadfile(plugin.dir .. '/lua/notebook_style/install.lua')()
        install.run(plugin)
      end,
      opts = {},
    }
<

Using vim.pack on Neovim 0.12+: >lua
    vim.pack.add({
      { src = 'https://github.com/stellarjmr/notebook_style.nvim' },
    })

    require('notebook_style').setup({})
<

With the optional inline execution backend installer, define the PackChanged
autocommand before vim.pack.add(): >lua
    local plugin_name = 'notebook_style.nvim'

    vim.api.nvim_create_autocmd('PackChanged', {
      callback = function(event)
        local data = event.data or {}
        local spec = data.spec or {}
        if spec.name ~= plugin_name then
          return
        end
        if data.kind ~= 'install' and data.kind ~= 'update' then
          return
        end

        local install = loadfile(data.path .. '/lua/notebook_style/install.lua')()
        install.run({ dir = data.path })
      end,
    })

    vim.pack.add({
      { src = 'https://github.com/stellarjmr/notebook_style.nvim', name = plugin_name, version = 'v0.7.0' },
    })

    require('notebook_style').setup({})
<

The backend install hooks download a prebuilt notebook-style-core backend on
supported tagged releases. Development branches, unsupported platforms, or
failed downloads fall back to building core/Cargo.toml with Cargo. You can
also run :NotebookStyleDownloadBackend manually.

Using packer.nvim: >lua
    use {
      'stellarjmr/notebook_style.nvim',
      ft = 'python',
      config = function()
        require('notebook_style').setup()
      end
    }
<

3. USAGE                                               *notebook_style-usage*

The plugin automatically activates for Python files with # %% delimiters.

.ipynb notebooks are supported when Jupytext is installed and configured so
Neovim edits the notebook as percent-style Python with # %% cell delimiters.
notebook_style.nvim works on that Python buffer; it does not parse raw .ipynb
JSON directly. If your Jupytext integration uses a filetype other than python,
add that filetype to the filetypes setup option.

Cell format: >python
    # %% Data Loading
    # This is a cell with a name
    import numpy as np
    print("Hello")

    # %% Processing
    # Another cell with a name
    x = [1, 2, 3]

    # %%
    # Cell without a name
    y = x * 2
<

The plugin will render borders around each cell, hiding the # %% delimiters
in normal/visual modes and showing them in insert mode.

Cell names (text after # %%) are automatically extracted and displayed in the
cell's top border (e.g., " #1 Data Loading").

4. COMMANDS                                         *notebook_style-commands*

                                                    *:NotebookStyleEnable*
:NotebookStyleEnable
    Enable the plugin for the current buffer.

                                                    *:NotebookStyleDisable*
:NotebookStyleDisable
    Disable the plugin for the current buffer.

                                                    *:NotebookStyleToggle*
:NotebookStyleToggle
    Toggle the plugin for the current buffer.

                                                    *:NotebookStyleRender*
:NotebookStyleRender
    Show or re-render cells for the current buffer. In manual render mode this
    is an explicit render action; in auto mode it also shows rendering again if
    it was hidden.

                                                    *:NotebookStyleToggleRender*
:NotebookStyleToggleRender
    Toggle cell rendering visibility for the current buffer. Rendering is shown
    automatically by default; use this command to hide it, and run it again to
    show it. With manual_render enabled, buffers start hidden and the first
    toggle shows rendering.

                                                    *:NotebookStyleRunCell*
:NotebookStyleRunCell
    Run the current Python cell through the Rust backend and render text output
    and image/png output below the cell. Requires building
    core/notebook-style-core first. Ghostty/Kitty use the Kitty graphics
    protocol; unsupported terminals show a text placeholder.

    By default, auto_venv checks for a project-local .venv next to the current
    file or in a parent directory. If .venv/bin/python (or
    .venv/Scripts/python.exe on Windows) can import ipykernel, that Python is
    used directly as the kernel interpreter. This avoids creating a registered
    Jupyter kernelspec per project. Set auto_venv = false to always use
    kernel_name instead.

    Inside tmux, enable passthrough with:
        set -g allow-passthrough on
    notebook_style wraps Kitty graphics escapes automatically when TMUX is set.
    Set NOTEBOOK_STYLE_DISABLE_TMUX_PASSTHROUGH=1 to opt out.

    image.cols is a maximum width. The plugin reads PNG dimensions and narrows
    the virtual placement when needed so normal plots are not centered inside
    an over-wide placeholder box. Adjust image.cell_height_to_width if your
    terminal font has an unusual cell aspect ratio.

                                                 *:NotebookStyleOpenOutput*
:NotebookStyleOpenOutput
    Open the current cell's captured output text in a focusable readonly
    floating scratch buffer. Use normal buffer navigation, search, Visual
    selection, yank, and scrolling for long output. This buffer is created only
    when the command is run; normal inline rendering remains virtual and
    lightweight.

                                                    *:NotebookStyleRunFile*
:NotebookStyleRunFile
    Run every non-empty Python cell in the current buffer.

                                           *:NotebookStyleRunCellAndMove*
:NotebookStyleRunCellAndMove
    Run the current Python cell and move the cursor to the next cell.

                                                    *:NotebookStyleKernelStart*
:NotebookStyleKernelStart
    Start the Python Jupyter kernel for the current buffer.

                                                    *:NotebookStyleKernelStop*
:NotebookStyleKernelStop
    Stop the Python Jupyter kernel for the current buffer.

                                               *:NotebookStyleKernelInterrupt*
:NotebookStyleKernelInterrupt
    Interrupt the running kernel for the current buffer, stopping a
    long-running cell (like pressing Ctrl-C in a terminal session). Mapped to
    <leader>ri by default.

                                                 *:NotebookStyleKernelRestart*
:NotebookStyleKernelRestart
    Restart the kernel for the current buffer: stop it, clear busy cell
    statuses, and start a fresh kernel using the same kernel resolution
    (auto_venv or kernel_name). If no kernel is running, this simply starts
    one.

                                             *:NotebookStyleDownloadBackend*
:NotebookStyleDownloadBackend
    Download the prebuilt notebook-style-core backend for the current tagged
    release, or fall back to cargo build --release. Use this after install or
    update if the inline execution backend is missing.

                                                       *notebook_style-health*
:checkhealth notebook_style
    Diagnose the plugin setup. Reports the Neovim version, whether setup() has
    been called, backend binary presence and executability, Jupyter kernel
    availability (project-local .venv detection for auto_venv and kernelspec
    lookup for kernel_name), and Kitty graphics support for inline images
    (including the tmux allow-passthrough setting when inside tmux).

    While a cell is executing, the output divider below the cell shows Out[*]
    and a "running…" line until the first output arrives. Statuses are reset
    when the kernel is stopped or restarted.

5. CONFIGURATION                               *notebook_style-configuration*

Default configuration: >lua
    require('notebook_style').setup({
      -- Cell delimiter pattern (Lua pattern)
      cell_delimiter = "^#%s*%%%%",

      -- Border style: 'solid', 'dashed', or 'double'
      border_style = 'solid',

      -- Colors
      colors = {
        border = '#6272A4',      -- Border color
        delimiter = '#50FA7B',   -- Delimiter marker color
        output = '#A9B1D6',      -- Inline output color
        result = '#C0CAF5',      -- Inline result color
        error = '#F7768E',       -- Inline error color
      },

      -- Visibility options
      hide_delimiter = true,           -- Hide # %% in normal/visual modes
      hide_border_in_insert = true,    -- Hide borders in insert mode

      -- Cell marker (shown in the top border when delimiter is hidden)
      cell_marker = ' ',              -- Python nerd font icon

      -- Cell name display options
      show_cell_name = true,           -- Show cell name from delimiter
      show_cell_number = true,         -- Show cell number in label
      cell_name_pattern = '^#%s*%%%%%s*(.-)%s*$',  -- Pattern to extract name
      cell_name_max_length = 40,       -- Max length for cell name
      cell_label_format_named = '{icon}#{number} {name}',    -- With name
      cell_label_format_unnamed = '{icon}#{number}',         -- Without name

      -- Legacy cell width configuration. Borders now use the full text area
      -- for stable wrapped-line rendering.
      cell_width_percentage = 80,      -- Cell width as % of window
      min_cell_width = 40,             -- Minimum cell width
      max_cell_width = 120,            -- Maximum cell width

      -- Inline execution options
      backend_cmd = nil,               -- Auto-detect notebook-style-core
      kernel_name = 'python3',         -- Jupyter kernelspec name
      auto_venv = true,                -- Prefer local .venv with ipykernel
      auto_start_kernel = true,        -- Start kernel on first run
      output_max_lines = 200,          -- Truncate very large outputs
      image = {
        rows = 18,                     -- Height for image/png outputs
        cols = 60,                     -- Max width for image/png outputs
        cell_height_to_width = 2.0,     -- Approximate terminal cell ratio
      },
      output_view = {
        width = 0.5,                    -- Fraction of editor columns
        height = 0.5,                   -- Fraction of editor rows
      },

      -- Default keymaps. Set a mapping to false to disable it.
      keymaps = {
        toggle_render = '<leader>rs',
        run_cell = '<leader>rr',
        run_file = '<leader>rf',
        run_cell_and_move = '<leader>rn',
        open_output = '<leader>ro',
        interrupt_kernel = '<leader>ri',
        restart_kernel = false,  -- Disabled by default
      },

      -- Render automatically by default. Set true to start hidden and render
      -- explicitly with :NotebookStyleRender or :NotebookStyleToggleRender.
      manual_render = false,

      -- Filetypes to enable the plugin for
      filetypes = { 'python' },
    })
<

                                                *notebook_style-border-style*
Border Styles~

Available border styles:
  - 'solid': Standard box drawing characters (default)
  - 'dashed': Dashed box drawing characters
  - 'double': Double-line box drawing characters

                                        *notebook_style-cell-marker*
Cell Marker~

Customize the text shown in the top border when # %% delimiters are hidden: >lua
    -- With Python nerd font icon (requires Nerd Font)
    cell_marker = ' Cell'  --  is Python icon

    -- Other examples
    cell_marker = '📘 Cell'    -- Book emoji
    cell_marker = '▶ Cell'     -- Play icon
    cell_marker = '# Cell'     -- Simple hash
    cell_marker = ''          -- Just the Python icon
<

Note: Nerd font icons require a Nerd Font to be installed.
See: https://www.nerdfonts.com/

                                            *notebook_style-cell-name*
Cell Names~

Display cell names extracted from delimiter lines. When you write: >python
    # %% My Cell Name
<
The plugin will display " #1 My Cell Name" in the top border instead of just
" #1".

Configuration options:
  - show_cell_name: Enable/disable cell name display (default: true)
  - show_cell_number: Enable/disable cell number display (default: true)
  - cell_name_pattern: Lua pattern to extract cell name from delimiter
  - cell_name_max_length: Maximum display length (default: 40)
  - cell_label_format_named: Format string when cell has a name
  - cell_label_format_unnamed: Format string when cell has no name

Format placeholders:
  - {icon}: The cell_marker icon
  - {number}: The cell number
  - {name}: The extracted cell name

Examples: >lua
    -- Show only name, no number
    cell_label_format_named = '{icon}{name}'
    cell_label_format_unnamed = '{icon}Unnamed'

    -- Custom format
    cell_label_format_named = 'Cell {number}: {name}'
<

                                            *notebook_style-cell-width*
Cell Width~

Control cell width with these options:
  - cell_width_percentage: Percentage of window width (1-100)
  - min_cell_width: Minimum width in characters
  - max_cell_width: Maximum width in characters

Examples: >lua
    -- Narrow cells (60% width)
    cell_width_percentage = 60

    -- Wide cells (95% width)
    cell_width_percentage = 95
<

6. DEVELOPMENT TESTS                                  *notebook_style-tests*

From the repository root, run: >sh
    tests/run_all.sh
<

The script checks Rust formatting, runs cargo test, builds the release backend,
and runs headless Neovim Lua smoke tests. The Lua suite exercises setup,
delimiter detection, rendering extmarks, and auto_venv when python3 can import
ipykernel. If ipykernel is unavailable, the auto_venv integration check is
reported as skipped.
